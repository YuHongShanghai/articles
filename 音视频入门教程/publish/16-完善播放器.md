## 16.1 进度条和时间显示

### 16.1.1 使用 SDL2 绘制简易进度条

```cpp
void draw_progress_bar(SDL_Renderer* renderer, int win_w, int win_h,
                        double current_time, double total_duration) {
    if (total_duration <= 0) return;

    // 进度条区域
    const int bar_height = 6;
    const int bar_margin = 20;
    const int bar_y = win_h - bar_height - 10;
    const int bar_w = win_w - 2 * bar_margin;

    // 进度比例
    double progress = current_time / total_duration;
    progress = std::max(0.0, std::min(1.0, progress));
    int filled_w = static_cast<int>(bar_w * progress);

    // 背景（深灰）
    SDL_SetRenderDrawColor(renderer, 60, 60, 60, 180);
    SDL_Rect bg_rect = {bar_margin, bar_y, bar_w, bar_height};
    SDL_RenderFillRect(renderer, &bg_rect);

    // 已播放部分（白色）
    SDL_SetRenderDrawColor(renderer, 255, 255, 255, 220);
    SDL_Rect fill_rect = {bar_margin, bar_y, filled_w, bar_height};
    SDL_RenderFillRect(renderer, &fill_rect);

    // 当前位置指示器（圆点效果用矩形模拟）
    SDL_SetRenderDrawColor(renderer, 255, 255, 255, 255);
    int dot_size = 12;
    SDL_Rect dot_rect = {
        bar_margin + filled_w - dot_size / 2,
        bar_y - (dot_size - bar_height) / 2,
        dot_size, dot_size
    };
    SDL_RenderFillRect(renderer, &dot_rect);
}
```

```cpp
case SDL_MOUSEBUTTONDOWN:

        int bar_y = win_h - 16;
        if (mouse_y >= bar_y - 10 && mouse_y <= win_h) {
            // 计算目标时间
            double ratio = static_cast<double>(mouse_x - 20) / (win_w - 40);
            ratio = std::max(0.0, std::min(1.0, ratio));
            double total_duration = ctx.fmt_ctx->duration / (double)AV_TIME_BASE;
            double target_time = ratio * total_duration;
            request_seek(&ctx, target_time);
        }
    }
    break;
```

## 16.2 窗口缩放与画面自适应

### 16.2.1 保持宽高比

```cpp
SDL_Rect calculate_display_rect(int win_w, int win_h, int video_w, int video_h) {
    float video_aspect = static_cast<float>(video_w) / video_h;
    float window_aspect = static_cast<float>(win_w) / win_h;

    SDL_Rect rect;
    if (video_aspect > window_aspect) {
        // 视频更宽，以窗口宽度为准，上下留黑边
        rect.w = win_w;
        rect.h = static_cast<int>(win_w / video_aspect);
        rect.x = 0;
        rect.y = (win_h - rect.h) / 2;
    } else {
        // 视频更高，以窗口高度为准，左右留黑边
        rect.h = win_h;
        rect.w = static_cast<int>(win_h * video_aspect);
        rect.x = (win_w - rect.w) / 2;
        rect.y = 0;
    }
    return rect;
}
```

### 16.2.2 渲染时使用计算的区域

```cpp
// 获取当前窗口大小
int win_w, win_h;
SDL_GetWindowSize(ctx.window, &win_w, &win_h);

// 计算保持宽高比的显示区域
SDL_Rect display_rect = calculate_display_rect(
    win_w, win_h,
    ctx.video_codec_ctx->width, ctx.video_codec_ctx->height);

// 清空（黑色背景，黑边）
SDL_SetRenderDrawColor(ctx.renderer, 0, 0, 0, 255);
SDL_RenderClear(ctx.renderer);

// 在计算的区域内渲染视频
SDL_RenderCopy(ctx.renderer, ctx.texture, nullptr, &display_rect);

double current = ctx.audio_clock.get();
double total = ctx.fmt_ctx->duration / (double)AV_TIME_BASE;
draw_progress_bar(ctx.renderer, win_w, win_h, current, total);

// 呈现
SDL_RenderPresent(ctx.renderer);
```

## 16.3 错误处理完善

### 16.3.1 统一的错误处理宏

```cpp
#define FF_CHECK(call, msg) do {                              \
    int _ret = (call);                                        \
    if (_ret < 0) {                                           \
        char _errbuf[AV_ERROR_MAX_STRING_SIZE];               \
        av_strerror(_ret, _errbuf, sizeof(_errbuf));          \
        std::cerr << msg << ": " << _errbuf << std::endl;    \
        goto cleanup;                                         \
    }                                                         \
} while(0)

// 使用
FF_CHECK(avformat_open_input(&fmt_ctx, filename, nullptr, nullptr),
         "无法打开文件");
FF_CHECK(avformat_find_stream_info(fmt_ctx, nullptr),
         "无法获取流信息");
```

### 16.3.2 RAII 资源管理类

```cpp
// 使用 unique_ptr 自动管理 FFmpeg 资源
template<typename T, void(*Deleter)(T**)>
struct FFmpegDeleter {
    void operator()(T* ptr) {
        if (ptr) {
            T* temp = ptr;
            Deleter(&temp);
        }
    }
};

using FormatContextPtr = std::unique_ptr<AVFormatContext,
    FFmpegDeleter<AVFormatContext, [](AVFormatContext** p){ avformat_close_input(p); }>>;

// 更简单的做法：使用 scope guard
class ScopeGuard {
    std::function<void()> func_;
public:
    explicit ScopeGuard(std::function<void()> func) : func_(std::move(func)) {}
    ~ScopeGuard() { if (func_) func_(); }
    ScopeGuard(const ScopeGuard&) = delete;
};

// 使用
AVFormatContext* fmt_ctx = nullptr;
avformat_open_input(&fmt_ctx, ...);
ScopeGuard guard([&]{ avformat_close_input(&fmt_ctx); });
```

## 16.4 性能优化建议

### 16.4.1 多线程解码

FFmpeg 解码器本身支持多线程：

```cpp
// 在打开解码器之前设置
codec_ctx->thread_count = 4;  // 使用 4 个线程解码
codec_ctx->thread_type = FF_THREAD_FRAME;  // 帧级别多线程
// 或
codec_ctx->thread_type = FF_THREAD_SLICE;  // Slice 级别多线程
```

### 16.4.2 硬件加速解码（简介）

FFmpeg 支持多种硬件加速方案：

| 平台 | 硬件加速 | API |
| --- | --- | --- |
| Linux | VAAPI / VDPAU | `h264_vaapi`, `hevc_vaapi` |
| Windows | DXVA2 / D3D11VA | `h264_dxva2` |
| macOS | VideoToolbox | `h264_videotoolbox` |
| NVIDIA | NVDEC / CUVID | `h264_cuvid` |

基本使用流程：

```cpp
// 查找硬件加速配置
for (int i = 0;; i++) {
    const AVCodecHWConfig* config = avcodec_get_hw_config(codec, i);
    if (!config) break;

    if (config->methods & AV_CODEC_HW_CONFIG_METHOD_HW_DEVICE_CTX) {
        // 找到支持的硬件加速方式
        AVBufferRef* hw_device_ctx = nullptr;
        av_hwdevice_ctx_create(&hw_device_ctx, config->device_type,
                               nullptr, nullptr, 0);
        codec_ctx->hw_device_ctx = av_buffer_ref(hw_device_ctx);
        break;
    }
}
```

硬件解码输出的帧格式（如 NV12）可能需要额外转换才能在 SDL2 中渲染。本教程不深入实现，留作进阶探索。

### 16.4.3 避免 CPU 格式转换

- 使用 SDL_UpdateYUVTexture 直接渲染 YUV 数据
- 避免每帧都调用 sws_scale（除非必要）
- 如果必须转换，复用 SwsContext

```
                                                                              │
                                                                         音频时钟
                                                                              │

```
