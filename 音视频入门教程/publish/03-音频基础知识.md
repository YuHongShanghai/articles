# 【从零手写播放器】第 3 章：音频基础知识

> 视频播放器不仅要"看得见"，还要"听得见"。本章我们来学习音频的核心概念：声音的数字化、采样率、位深、PCM 数据格式，以及常见的音频编码标准。

## 3.1 声音的本质

声音是一种**机械波**，通过空气（或其他介质）传播。声波的关键物理参数：

- **频率（Hz）**：每秒振动次数，决定音调高低。人耳能听到 20Hz~20000Hz
- **振幅**：波形的高度，决定音量大小
- **波形**：波的形状，决定音色（同样音高的钢琴和吉他听起来不同）

![声波的三个关键物理参数](https://gitee.com/yuhong1234/ffmpeg-player-tutorial/raw/master/images/diagram-03-sound-wave.png)

## 3.2 声音的数字化：采样与量化

要在计算机中处理声音，需要将连续的模拟信号转换为离散的数字信号。这个过程分为三步：

### 3.2.1 采样（Sampling）

在时间轴上以固定间隔"取点"，记录每个时刻的声波幅度值。

![采样过程示意图](https://gitee.com/yuhong1234/ffmpeg-player-tutorial/raw/master/images/diagram-03-sampling.png)

**采样率（Sample Rate）** 就是每秒采样的次数，单位是 Hz：

| 采样率 | 应用场景 | 可记录频率上限 |
| --- | --- | --- |
| 8000 Hz | 电话 | 4000 Hz |
| 22050 Hz | 低质量音频 | 11025 Hz |
| **44100 Hz** | CD 音质 | 22050 Hz |
| **48000 Hz** | 视频/专业音频 | 24000 Hz |
| 96000 Hz | 高保真音频 | 48000 Hz |

> **奈奎斯特定理**：采样率必须至少是信号最高频率的 2 倍，才能无失真地还原原始信号。人耳最高能听到约 20000Hz，所以 44100Hz 采样率就足够覆盖人耳的听觉范围。

### 3.2.2 量化（Quantization）

将采样得到的连续幅度值"取整"到有限个离散的级别上。

**位深（Bit Depth）** 决定了量化的精度：

| 位深 | 量化级别数 | 动态范围 | 应用 |
| --- | --- | --- | --- |
| 8 bit | 256 级 | 48 dB | 早期电话 |
| **16 bit** | 65536 级 | 96 dB | CD 音质 |
| 24 bit | 16777216 级 | 144 dB | 专业录音 |
| 32 bit float | 连续 | >1500 dB | 音频处理内部格式 |

![量化精度对比](https://gitee.com/yuhong1234/ffmpeg-player-tutorial/raw/master/images/diagram-03-quantization.png)

位深越高，能表示的音量细节越丰富，量化噪声越小。

### 3.2.3 编码（Encoding）

将量化后的数字值按约定格式存储，最直接的存储方式就是 **PCM（Pulse Code Modulation，脉冲编码调制）**。

## 3.3 PCM 原始音频数据

PCM 是未经压缩的原始音频数据格式，就是直接按顺序存储每一个采样点的量化值。

### 3.3.1 PCM 数据布局

PCM 数据的描述需要三个参数：**采样率、位深、声道数**。

![PCM 数据存储布局](https://gitee.com/yuhong1234/ffmpeg-player-tutorial/raw/master/images/diagram-03-pcm-layout.png)

- **单声道（Mono）**：采样点按时间顺序依次排列
- **双声道交错存储（Interleaved / Packed）**：左右声道采样点交替排列 `[L1][R1][L2][R2]...`
- **双声道平面存储（Planar）**：每个声道的数据连续存储 `[L1][L2]...[R1][R2]...`

> FFmpeg 中这两种格式分别用 `AV_SAMPLE_FMT_S16`（交错）和 `AV_SAMPLE_FMT_S16P`（平面）等来标识。后缀带 `P` 的就是平面格式。

### 3.3.2 PCM 数据量计算

```
数据量 = 采样率 × 位深(字节) × 声道数 × 时长

例：CD 音质（44100Hz, 16bit, 双声道），1 分钟：
数据量 = 44100 × 2 × 2 × 60 = 10,584,000 字节 ≈ 10 MB

1 小时 ≈ 635 MB
```

原始 PCM 数据量同样非常大，所以需要音频编码压缩。

## 3.4 声道与声道布局

| 声道配置 | 声道数 | 说明 |
| --- | --- | --- |
| Mono | 1 | 单声道 |
| Stereo | 2 | 立体声（左/右） |
| 5.1 | 6 | 环绕声（前左/前右/中置/低音/后左/后右） |
| 7.1 | 8 | 增强环绕声 |

![常见声道布局](https://gitee.com/yuhong1234/ffmpeg-player-tutorial/raw/master/images/diagram-03-channel-layout.png)

在 FFmpeg 中，声道布局用 `AVChannelLayout` 结构体描述（FFmpeg 5.1+），常见的有：

- `AV_CHANNEL_LAYOUT_MONO`
- `AV_CHANNEL_LAYOUT_STEREO`
- `AV_CHANNEL_LAYOUT_5POINT1`

## 3.5 音频帧

与视频类似，音频数据也按"帧"处理。但音频帧的概念和视频帧不同：

- **视频帧**：一张完整的画面
- **音频帧**：固定数量的采样点（由编码器决定）

| 编码器 | 每帧采样数 | 常用采样率下每帧时长 |
| --- | --- | --- |
| AAC | 1024 | ≈ 23.2 ms（44100Hz） |
| MP3 | 1152 | ≈ 26.1 ms（44100Hz） |
| Opus | 960（可变） | ≈ 20 ms（48000Hz） |
| PCM（无编码） | 无固定帧大小 | — |

![音频帧概念图](https://gitee.com/yuhong1234/ffmpeg-player-tutorial/raw/master/images/diagram-03-audio-frame.png)

在 FFmpeg 中，一个音频 `AVFrame` 包含：
- `nb_samples`：该帧的采样点数
- `sample_rate`：采样率
- `format`：采样格式（如 AV_SAMPLE_FMT_FLTP）
- `ch_layout`：声道布局
- `data[]`：实际的音频数据

## 3.6 常见音频编码标准

| 编码 | 类型 | 特点 |
| --- | --- | --- |
| PCM | 无损/无压缩 | 数据量大，质量最好 |
| FLAC | 无损压缩 | 压缩约 50%，完美还原 |
| MP3 | 有损压缩 | 128kbps 即可听，兼容性最好 |
| **AAC** | 有损压缩 | 同码率下质量优于 MP3，目前主流 |
| **Opus** | 有损压缩 | 低延迟优势，适合实时通信 |
| Vorbis | 有损压缩 | 开源，用于 WebM/OGG 容器 |

![音频编码文件大小对比](https://gitee.com/yuhong1234/ffmpeg-player-tutorial/raw/master/images/diagram-03-audio-compression.png)

对于视频文件来说，最常搭配的音频编码是 **AAC**（常用于 MP4 容器）。

## 3.7 Demo：生成 PCM 音频并理解采样率

### 3.7.1 用 FFmpeg 生成正弦波音频

```bash
# 生成一个 440Hz（标准 A 音）的正弦波，时长 3 秒
# 采样率 44100Hz，16位，单声道，输出为原始 PCM 文件
ffmpeg -y -f lavfi -i "sine=frequency=440:duration=3:sample_rate=44100" \
  -f s16le -acodec pcm_s16le output_44100.pcm

# 同样的声音，但采样率只有 8000Hz
ffmpeg -y -f lavfi -i "sine=frequency=440:duration=3:sample_rate=8000" \
  -f s16le -acodec pcm_s16le output_8000.pcm
```

### 3.7.2 查看 PCM 文件大小

```bash
ls -la output_*.pcm
```

预期输出：

```
-rw-r--r-- 1 user user 264600 output_44100.pcm   # 44100 × 2字节 × 1声道 × 3秒
-rw-r--r-- 1 user user  48000 output_8000.pcm    # 8000 × 2字节 × 1声道 × 3秒
```

数据量之比正好等于采样率之比：264600 / 48000 ≈ 5.5 ≈ 44100 / 8000。

### 3.7.3 播放 PCM 文件

PCM 是原始数据，没有文件头，播放时必须指定参数：

```bash
# 播放 44100Hz 采样率的 PCM
ffplay -f s16le -ar 44100 -ch_layout mono output_44100.pcm

# 播放 8000Hz 采样率的 PCM
ffplay -f s16le -ar 8000 -ch_layout mono output_8000.pcm
```

> **注意**：如果你使用的是 FFmpeg 7.x 之前的版本，需要将 `-ch_layout mono` 替换为 `-ac 1`。FFmpeg 7.0 起移除了 `-ac` 选项，改用 `-ch_layout` 来指定声道布局。

两个文件播放出来都是 440Hz 的"嘟"声，但 8000Hz 采样率的版本听起来会稍微"闷"一些（高频细节被截断了）。

### 3.7.4 故意用错参数播放

```bash
# 用 22050Hz 的采样率播放 44100Hz 的文件
ffplay -f s16le -ar 22050 -ch_layout mono output_44100.pcm
```

你会发现声音变低了一个八度，播放时长变成了 6 秒——因为播放器以一半的速度"消耗"采样点。

```bash
# 用 88200Hz 的采样率播放 44100Hz 的文件
ffplay -f s16le -ar 88200 -ch_layout mono output_44100.pcm
```

声音变高了一个八度，播放时长只有 1.5 秒——播放速度翻倍了。

这个实验直观地说明了**采样率的含义**：它告诉播放器每秒应该消耗多少个采样点。

### 3.7.5 编码为 AAC 并对比

```bash
# 将 PCM 编码为 AAC（128kbps）
ffmpeg -y -f s16le -ar 44100 -ch_layout mono -i output_44100.pcm \
  -c:a aac -b:a 128k output.aac

# 对比大小
ls -la output_44100.pcm output.aac
```

预期结果：

```
264600  output_44100.pcm   # PCM 原始数据
~50000  output.aac         # AAC 压缩后（约为原来的 1/5）
```

AAC 压缩后的文件远小于原始 PCM，但听感几乎没有差别。这就是有损压缩的威力。

## 小结

本章我们学习了音频的核心基础知识：

1. **声音数字化**：采样（时间轴离散化） + 量化（幅度离散化）
2. **核心参数**：采样率（Hz）、位深（bit）、声道数
3. **PCM 格式**：未压缩的原始音频数据，交错存储 vs 平面存储
4. **音频帧**：每帧包含固定数量的采样点
5. **音频编码**：AAC 是目前视频中最常用的音频编码

在后续的播放器开发中，我们需要：
- 用 FFmpeg 将音频解码为 PCM 数据
- 将 PCM 数据送入 SDL2 的音频系统播放
- 基于音频的播放进度来同步视频
